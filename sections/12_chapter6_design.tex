    \chapter{Design}

    \section{Use Case Diagram}
    Primary actors are Voter, Election Officer, and System Auditor. Key use cases include voter enrollment validation, biometric capture, cast vote, verify ledger integrity, and generate tally.

    Use case description summary:
    The voting session begins only after the candidate list is available and the device stack is active or running in simulation mode. The main flow moves from identifier entry to biometric capture, then to vote casting and encrypted storage. If the hardware is unavailable, the system falls back to simulation mode with audit flags so the workflow can continue. Once the cast is completed, the vote is recorded, the cast registry is updated, and the audit event is persisted.

    \section{Class Diagram}
    The class model centers on clean architecture boundaries:
    The core domain and use cases are implemented in domain.py, usecases.py, and state_machine.py. Port interfaces in ports.py define the contracts for storage, audit, chain anchoring, and devices, while adapter implementations such as storage_fernet_hashchain.py, audit_log_hashchain.py, devices_mock.py, and chain_simulated.py connect the core logic to the external environment.

    \section{Entity-Relationship (ER) Diagram}
    The prototype uses file-ledger entities instead of relational tables:
    The BallotRecord stores the sequence number, encrypted payload, previous hash, current hash, and timestamp. The AuditRecord stores the event type, metadata, previous hash, current hash, and timestamp. The CastRegistry stores the salted identifier hash, cast timestamp, and session metadata.

    \section{Deployment Diagram}
    Deployment is designed for polling-station edge execution:
    The deployment model uses a Windows or edge compute unit running the UI and core services. Camera and biometric peripherals are connected locally, and ballots together with audit logs are stored in encrypted form on the same site. An optional network uplink remains available for chain anchoring and centralized monitoring.
