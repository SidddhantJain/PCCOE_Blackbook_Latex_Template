\chapter{Methodology}

\section{Introduction}
VoteGuard Pro follows a security-first and accessibility-constrained methodology. The method combines modular software engineering, threat-informed design, and prototype validation.

The selected approach was chosen because it allows independent verification of critical layers (UI, storage, audit, and optional chain anchoring) while supporting rapid iteration in academic constraints.

\section{System Architecture}
The architecture follows clean-separation principles:
\begin{itemize}
\item \textbf{Core Layer:} domain models, ports, use cases, and state machine.
\item \textbf{Adapter Layer:} encrypted storage adapter, audit log adapter, chain simulation, and device adapters.
\item \textbf{UI Layer:} Aadhaar entry, biometric capture, and voting screen workflow.
\item \textbf{Optional ML Layer:} overlays and anomaly hints for operator observability only.
\end{itemize}

Data flow:
\begin{enumerate}
\item Capture voter identifiers and initialize voting session.
\item Perform biometric capture/face detection and continue guided flow.
\item Accept candidate selection and confirmation.
\item Encrypt ballot payload and append to hash-chained ledger.
\item Record audit event and optionally prepare chain-anchoring metadata.
\end{enumerate}

\section{Algorithm}
Key algorithms and controls used:
\begin{itemize}
\item \textbf{Encrypted Append Algorithm:} encrypt vote payload via Fernet, append as next sequence element, and update chain hash.
\item \textbf{Cast-Registry Guard:} create salted identifier hash and reject duplicate cast attempts.
\item \textbf{Ledger Verification Algorithm:} traverse records, validate sequence continuity and previous-hash integrity.
\item \textbf{UI State Progression:} deterministic screen transitions to prevent overlapping dialogs and race conditions.
\end{itemize}

Representative pseudocode:
\begin{verbatim}
Input: voter_id, candidate
if cast_registry.contains(hash_salt(voter_id)):
	reject_vote("duplicate attempt")
else:
	payload = encrypt({candidate, timestamp, station_id})
	record = append_hash_chain(payload)
	audit_log.append("vote_cast", record.hash)
	cast_registry.add(hash_salt(voter_id))
	confirm_success()
\end{verbatim}

\subsection{Detailed Architecture Layers}

The system is organized in six layered tiers following clean architecture principles:

\begin{enumerate}
\item \textbf{Presentation Layer} - Voter UI (tablet/kiosk), Election Officer Dashboard, System Auditor Console, REST API endpoints.
\item \textbf{Application Layer} - Session Manager, Vote Casting Orchestrator, Enrollment Validator, Verification Engine, Tally Calculator.
\item \textbf{Domain Layer} - Voter Entity, Ballot Entity, Cast Registry, Cryptography Service, State Machine.
\item \textbf{Port Interfaces} - IStorage, IAudit, IChainAnchor, IDevice (dependency inversion via abstraction).
\item \textbf{Adapter Layer} - StorageHashChainAdapter, AuditLogHashChainAdapter, ChainAnchorServiceAdapter, DeviceMockAdapter.
\item \textbf{Infrastructure Layer} - Encrypted File Store, Audit Records, Biometric Scanner, Card Reader, Printer, Blockchain Service, Remote Verification.
\end{enumerate}
